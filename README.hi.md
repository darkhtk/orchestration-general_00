# Claude Orchestration

**[English](README.md) | [한국어](README.ko.md) | [日本語](README.ja.md) | [中文](README.zh.md) | [Español](README.es.md) | [Deutsch](README.de.md) | [Français](README.fr.md) | [हिन्दी](README.hi.md) | [ไทย](README.th.md) | [Tiếng Việt](README.vi.md)**

फाइल-आधारित एसिंक्रोनस संवाद के माध्यम से गेम डेवलपमेंट पर सहयोग करने वाले एकाधिक Claude CLI एजेंट।

एक बैट फाइल सब कुछ सेटअप कर देती है। एजेंट स्वायत्त रूप से कार्य उठाते हैं, फीचर लागू करते हैं, कोड की समीक्षा करते हैं, और प्रोजेक्ट बोर्ड प्रबंधित करते हैं — सब कुछ मार्कडाउन फाइलों के माध्यम से समन्वित।

## यह कैसे काम करता है

```
orchestrate.bat  (डबल-क्लिक करें)
    |
    |-- डिपेंडेंसी जांच (Git, Claude CLI)
    |-- गेम प्रोजेक्ट फोल्डर चुनें (आधुनिक डायलॉग)
    |-- इंजन का स्वतः पता लगाएं (Unity / Godot / Unreal)
    |-- इंटरैक्टिव सेटअप:
    |       Git रिमोट, कमिट पॉलिसी, डेव दिशा,
    |       एजेंट मोड, रिव्यू स्तर, डॉक स्कैनिंग
    |-- प्रोजेक्ट कॉन्फिग + एजेंट प्रॉम्प्ट्स जनरेट करें
    |-- अलग-अलग टर्मिनलों में एजेंट लॉन्च करें
    v
  4 एजेंट समानांतर में चल रहे हैं, orchestration/ के माध्यम से संवाद
```

## एजेंट

| एजेंट | भूमिका | यह क्या करता है |
|-------|------|-------------|
| **Supervisor** | ऑर्केस्ट्रेटर | एसेट निर्माण, कोड गुणवत्ता ऑडिट, बग फिक्स, कार्य प्रबंधन |
| **Developer** | बिल्डर | गेम लॉजिक लागू करता है, टेस्ट लिखता है, कोड कमिट करता है |
| **Client** | समीक्षक | मल्टी-पर्सोना QA समीक्षा, गुणवत्ता फीडबैक |
| **Coordinator** | प्रबंधक | बोर्ड सिंक, बैकलॉग पुनःपूर्ति, स्पेक लेखन, एजेंट मॉनिटरिंग |

## त्वरित शुरुआत

```bash
# 1. क्लोन करें
git clone git@github.com:darkhtk/orchestration-general_00.git
cd orchestration-general_00

# 2. orchestrate.bat पर डबल-क्लिक करें
#    - आपका गेम प्रोजेक्ट फोल्डर चुनता है
#    - इंजन, डायरेक्टरी, मौजूदा डॉक्स का स्वतः पता लगाता है
#    - सेटअप प्रश्न पूछता है (दिशा, एजेंट मोड, आदि)
#    - एजेंट लॉन्च करता है

# या कमांड लाइन से:
orchestrate.bat "C:\path\to\your\game"
```

## सेटअप विकल्प

इंटरैक्टिव सेटअप यह पूछता है:

| विकल्प | चुनाव | डिफ़ॉल्ट |
|--------|---------|---------|
| **मौजूदा डॉक्स** | एजेंटों के पहले लूप में पढ़ने के लिए प्रोजेक्ट डॉक्स स्कैन करें | हां |
| **Git** | रिपो इनिशियलाइज़ करें, रिमोट URL सेट करें | स्वतः-पता |
| **कमिट/पुश पॉलिसी** | task / review / batch / manual | task |
| **डेव दिशा** | stabilize / feature / polish / content / custom | feature |
| **एजेंट मोड** | full (4) / lean (2) / solo (1) | full |
| **रिव्यू स्तर** | strict / standard / minimal | standard |

## क्या बनाया जाता है

जब आप किसी गेम प्रोजेक्ट पर orchestrate.bat चलाते हैं, तो यह बनाता है:

```
your-game-project/
  orchestration/
    project.config.md        # सभी सेटिंग्स (एजेंट हर लूप में यह पढ़ते हैं)
    BOARD.md                 # कानबान बोर्ड (Backlog > In Progress > In Review > Done)
    BACKLOG_RESERVE.md       # डेवलपर्स के लिए कार्य पूल
    agents/                  # एजेंट भूमिका परिभाषाएं
    prompts/                 # एजेंट लॉन्च प्रॉम्प्ट्स
    templates/               # दस्तावेज़ टेम्पलेट (task, review, spec, आदि)
    tasks/                   # कार्य स्पेक्स (TASK-001.md, ...)
    reviews/                 # समीक्षा परिणाम (REVIEW-001-v1.md, ...)
    decisions/               # सुपरवाइज़र निर्णय
    discussions/             # एजेंट चर्चाएं (एसिंक्रोनस बहस)
      concluded/             # समाधान हुई चर्चाएं
    specs/                   # फीचर स्पेक्स (SPEC-R-001.md, ...)
    logs/                    # प्रति-एजेंट लूप लॉग
    .run_SUPERVISOR.sh       # एजेंट रनर स्क्रिप्ट्स
    .run_DEVELOPER.sh
    .run_CLIENT.sh
    .run_COORDINATOR.sh
```

## वर्कफ़्लो

```
Backlog --> In Progress --> In Review --> Done
                ^               |
                '-- Rejected <--'
```

1. **Supervisor/Coordinator** BACKLOG_RESERVE में कार्य बनाते हैं
2. **Developer** शीर्ष कार्य उठाता है, उसे लागू करता है
3. Developer कार्य को In Review में ले जाता है
4. **Client** मल्टी-पर्सोना समीक्षा करता है (4 समीक्षक पर्सोना)
5. APPROVE -> Done / NEEDS_WORK -> Rejected -> Developer ठीक करता है

## एजेंट मोड

### Full (4 एजेंट)
सभी एजेंट सक्रिय। पूर्ण समीक्षा चक्र, बोर्ड प्रबंधन, एसेट निर्माण।

### Lean (2 एजेंट)
केवल Developer + Supervisor। कोई समर्पित समीक्षक या कोऑर्डिनेटर नहीं। Supervisor समीक्षा और बोर्ड सिंक संभालता है।

### Solo (1 एजेंट)
सभी भूमिकाओं को मर्ज किए गए एकल Developer एजेंट। स्व-समीक्षा, स्व-प्रबंधित बोर्ड। छोटे प्रोजेक्ट या एकल विकास के लिए अच्छा।

## पुनः शुरू करना

यदि आप किसी ऐसे प्रोजेक्ट पर orchestrate.bat चलाते हैं जिसमें पहले से `orchestration/` है, तो यह मौजूदा सेटअप का पता लगाता है:

```
  मौजूदा ऑर्केस्ट्रेशन पाया गया!
  मोड: full    दिशा: stabilize

  1) Resume      - केवल एजेंट लॉन्च करें (सेटअप छोड़ें)
  2) Reconfigure - सेटअप फिर से चलाएं
  3) Cancel
```

## अन्य टूल्स

| फाइल | यह क्या करता है |
|------|-------------|
| `add-feature.bat` | सादे टेक्स्ट में फीचर का वर्णन करें -> स्वतः कार्य + स्पेक्स जनरेट होते हैं |
| `monitor.bat` | Unity/Godot एडिटर लॉग में रनटाइम एरर की निगरानी करें, स्वतः बग कार्य बनाएं |

## मुख्य तंत्र

### FREEZE
BOARD.md के शीर्ष पर FREEZE नोटिस जोड़ें -> सभी एजेंट तुरंत रुक जाते हैं। पुनः शुरू करने के लिए इसे हटाएं।

### चर्चाएं
एजेंट `discussions/` में एसिंक्रोनस बहस खोल सकते हैं। डिज़ाइन निर्णयों, प्राथमिकता परिवर्तनों, प्रोटोकॉल सुधारों के लिए उपयोग किया जाता है। सभी एजेंट अपने अनुभाग में उत्तर देते हैं, फिर supervisor निष्कर्ष निकालता है।

### स्व-प्रगति
Developer बिना supervisor की प्रतीक्षा किए कार्यों में स्वतः आगे बढ़ सकता है। QA/बैलेंस कार्य समीक्षा को पूरी तरह छोड़ देते हैं। नए सिस्टम कार्यों के लिए हमेशा Client समीक्षा आवश्यक है।

## फाइल अवलोकन

```
orchestrate.bat          # मुख्य एंट्री पॉइंट (सेटअप + लॉन्च)
add-feature.bat          # टेक्स्ट विवरण से फीचर जोड़ें
monitor.bat              # रनटाइम एरर मॉनिटरिंग
pick-folder.ps1          # आधुनिक फोल्डर पिकर डायलॉग (IFileDialog COM)
auto-setup.sh            # इंजन पहचान, कॉन्फिग जनरेशन, इंटरैक्टिव सेटअप
init.sh                  # डायरेक्टरी संरचना निर्माण
launch.sh                # क्रॉस-प्लेटफॉर्म एजेंट लॉन्चर
extract-features.sh      # कोडबेस विश्लेषण -> FEATURES.md
seed-backlog.sh          # FEATURES.md -> कार्य + स्पेक्स
add-feature.sh           # प्राकृतिक भाषा -> कार्य + स्पेक्स
monitor.sh               # Editor.log वॉचर + एरर रिपोर्टर
project.config.md        # खाली कॉन्फिग टेम्पलेट
framework/
  agents/                # एजेंट भूमिका परिभाषाएं (4 फाइलें)
  prompts/               # एजेंट लूप प्रॉम्प्ट्स (4 फाइलें)
  templates/             # दस्तावेज़ टेम्पलेट (7 फाइलें)
sample-config/           # Unity/Godot के लिए उदाहरण कॉन्फिग
```
