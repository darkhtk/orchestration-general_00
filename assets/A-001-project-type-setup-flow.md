# A-001: 프로젝트 타입별 설정 플로우 다이어그램

## 개요
각 엔진/플랫폼(Unity, Godot, Unreal, Web, Python)별 auto-setup.sh 감지 흐름과 생성되는 설정 항목을 보여주는 플로우 다이어그램

## 플랫폼별 감지 조건

### Unity
```
검사 파일:
├── *.uproject               (Unreal과 구별)
├── Assets/                  (필수 디렉토리)
├── ProjectSettings/         (필수 디렉토리)
├── Library/                 (선택적)
└── Packages/manifest.json   (선택적)

생성 설정:
├── Unity 버전 (2021.3 LTS, 2022.3 LTS 등)
├── 타겟 플랫폼 (Windows, Android, iOS 등)
├── 렌더링 파이프라인 (Built-in, URP, HDRP)
├── 에셋 임포트 규칙
└── 빌드 자동화 설정
```

### Godot
```
검사 파일:
├── project.godot            (필수)
├── *.gd, *.cs              (스크립트)
├── .godot/                  (엔진 캐시)
└── export_presets.cfg       (선택적)

생성 설정:
├── Godot 버전 (3.x vs 4.x)
├── 스크립트 언어 (GDScript vs C#)
├── 렌더링 설정
├── 씬 구조 규칙
└── 익스포트 설정
```

### Unreal Engine
```
검사 파일:
├── *.uproject               (필수)
├── Source/                  (C++ 프로젝트)
├── Content/                 (에셋)
├── Config/                  (설정)
└── Binaries/                (빌드 결과물)

생성 설정:
├── Unreal 버전 (4.27, 5.0+ 등)
├── C++ 표준 (C++17, C++20)
├── 타겟 플랫폼
├── 패키징 설정
└── 테스트 자동화
```

### Web (React/Next.js)
```
검사 파일:
├── package.json             (필수)
├── tsconfig.json            (TypeScript)
├── next.config.js           (Next.js)
├── webpack.config.js        (Webpack)
└── src/                     (소스 코드)

생성 설정:
├── 프레임워크 (React, Next.js, Vue 등)
├── TypeScript 설정
├── 번들러 (Webpack, Vite 등)
├── 테스트 (Jest, Cypress 등)
└── 배포 설정
```

### Python
```
검사 파일:
├── requirements.txt         (pip)
├── pyproject.toml           (Poetry/PDM)
├── setup.py                 (setuptools)
├── Pipfile                  (Pipenv)
└── *.py                     (Python 스크립트)

생성 설정:
├── Python 버전 (3.8+)
├── 의존성 관리 도구
├── 가상환경 설정
├── 테스트 프레임워크
└── 린팅/포매팅 규칙
```

## 설정 우선순위
1. **명시적 플래그**: `--unity`, `--godot` 등
2. **자동 감지**: 위 플로우 순서대로
3. **기본값**: Generic 프로젝트 설정

## 충돌 해결
여러 플랫폼이 동시 감지되는 경우:
- Unity + Unreal → 사용자 선택 프롬프트
- Web + Python → package.json 우선
- 감지 실패 → Generic 설정 + 수동 구성 안내
